Architecture

Four pillars. One runtime.

ROSTR wraps the full Hermes Agent runtime with a structured intelligence layer — every input is compiled, classified, researched, and remembered.

Pillar 01

PAL — Prompt Abstraction Layer

The LLM Compiler. Transforms raw natural language into strictly-typed AgentManifests through a 5-stage pipeline: intent extraction, context injection, semantic enhancement, runtime compilation, output routing.

Pillar 02

NPAO — Decision Engine

Navigate, Prioritize, Allocate, Orchestrate. Classifies every task into a 5D phase (PreD/Design/Dev/Deploy/Debug) and computes a 4D priority score. Selects orchestration patterns automatically.

Pillar 03

RAG DAL — Knowledge Engine

Dynamic Acquisition Layer. Autonomous multi-pass retrieval with 3-tier source credibility scoring. Converges at 0.8 confidence. Max 4 passes. Gap-fills automatically on low coverage.

Pillar 04

Rostr Hub — Agent OS

Persistent multi-namespace state: session, project, organization, agent. Registers agents, logs decisions, compounds knowledge across sessions. The memory that makes agents improve over time.

Get Started

Install in 60 seconds.

One command. Works on macOS, Linux, and Termux (Android). Requires Python 3.11+. No root access needed.

# Clone and install ROSTR Agent git clone https://github.com/diamitani/rostr-agent.git cd rostr-agent ./setup-rostr.sh # Or install from PyPI pip install rostr-agent # Run interactive CLI rostr-agent # Or use individual commands hermes pal compile "build a REST API for user management" hermes npao classify "the auth endpoint is returning 403 for admin users" hermes ragdal search "best practices for rate limiting in production APIs" hermes hub compound
What the installer does

1. Environment Detection

Detects desktop/server vs Android/Termux. Adapts paths and package managers automatically.

2. Python via UV

Installs uv (Astral's fast Python manager), provisions Python 3.11 in an isolated venv.

3. Dependency Install

Installs all deps via uv pip in the venv. No system pollution. Fully reversible.

4. Environment Config

Creates .env from template. API keys stay local. Never committed to git.

5. CLI Symlink

Links the `hermes` and `rostr-agent` commands to ~/.local/bin for immediate use.

6. Setup Wizard

Optional interactive wizard configures your preferred model, provider, and delivery targets.

Security

Secure by default.

ROSTR inherits Hermes's security model and adds additional safeguards for the intelligence layer.

No telemetry

Zero data leaves your machine unless you explicitly configure an external provider. All state is local SQLite.

Secrets isolation

API keys live in .env only. Never embedded in code, never committed. .gitignore enforced on install.

Sandboxed execution

Tool calls are permission-gated. Destructive operations require explicit confirmation. Dry-run default for consequential actions.

Provider-agnostic

Run fully local via Ollama — zero external API calls. Or route through any of 20+ providers. Your choice.

Auditable decisions

Every NPAO classification, PAL compilation, and Hub entry is logged with timestamp and rationale. Full audit trail.

MIT License

Fully open source. Read every line. Fork it. Audit it. No vendor lock-in, no proprietary dependencies.

Why It Works

What ROSTR adds to Hermes.

Hermes is already excellent — a complete personal agent with memory, skills, multi-platform delivery, and 20+ provider support. ROSTR doesn't replace it. ROSTR gives it structure.

Without ROSTR

Raw prompts → raw responses

The model does its best with whatever you give it. No intent extraction. No priority awareness. No cross-session memory. No source verification.

Good model. No system thinking.
With ROSTR

Compiled prompts → structured execution

Every input is compiled into a typed manifest. Phase determines urgency. RAG DAL verifies claims. Hub remembers what worked. The agent improves with every session.

Same model. Structured intelligence layer.
The Key Insight

Models don't need better weights — they need better structure

A well-structured prompt with the right context, phase classification, and memory recall outperforms a raw prompt to a larger model. ROSTR proves this empirically.

Architecture > parameter count.
